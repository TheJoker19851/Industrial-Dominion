# ADMIN TASK

## Title
Make the starter economy produce real industrial trade-offs

## Goal
Move beyond a guided starter loop and begin shaping Industrial Dominion into a real economic strategy game by introducing meaningful trade-offs in market and production decisions.

## Product intent
The current starter loop is readable and technically validated, but it is still too shallow to feel like the intended game.

The next phase must make the player feel:
- market pressure
- production choice
- value-added transformation
- early industrial progression

## Current context
We already have:
- extractor placement
- processor placement
- processor-gated production
- inventory, buy, sell, transfer
- dashboard guidance and clarity improvements
- regression-validated starter loop

## Primary objective
Introduce the smallest high-value gameplay changes that make economic decisions feel more meaningful, starting with market and production depth rather than more dashboard polish.

## Priority order
1. Market depth and buy/sell trade-offs
2. Factory/production stage progression
3. Workforce constraints
4. Cartel foundations

## Hard constraints
- backend remains authoritative
- no large uncontrolled refactor
- no broad dashboard work unless directly needed
- no complex endgame systems yet
- EN + FR required
- mobile usability preserved

## Scope guidance
The orchestrator may generate small tasks around:
- stronger raw vs processed trade-offs
- one additional factory stage or production step
- minimal workforce/capacity constraint
- market decisions with clearer economic consequences

## Out of scope
- full cartel gameplay
- broad social/political systems
- major UI redesign
- complex simulation overhaul
- deep balancing framework

## Quality expectations
- small vertical slices
- visible gameplay value
- real strategic consequence
- minimal file churn
- safe iterative execution

## Execution instructions
- stop generating dashboard-only tasks
- generate the next tasks in market/production/workforce order
- prioritize the smallest change that adds real strategic depth
- stop after preparing the next task unless coding is requested
